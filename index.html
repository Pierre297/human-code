Human Code

Implementare un Layout assegnato come esercitazione

  1. Accedo a Slack per la lettura dell'esecizio.

    • SE il computer è spento:
      • Accendo il Computer
      • Inserisco Password per accedere al dekstop
      • Accedo a Slack

    • ALTRIMENTI:
      • Accedo a Slack da Dekstop

  2. Vado sul canale comunicazioni

    • SE l'account è collegato correttamente a Slack:
      • Accedo al canale testuale "comunicazioni"

    • ALTRIMENTI:
      • Accedo al profilo Slack inserendo E-mail e Password

  3. Leggo l'esercitazione della giornata

    • SE comprendo bene l'esercizio:
      • Posso creare una nuova repository.
      • Copio il nome della repo da incollare su Github.

    • ALTRIMENTI:
      • Accedo a Google Drive e consulto le slide della lezione della mattina per capire al meglio l'esercizio e i temi.

  4. Apro GitHub e creo una nuova repository

    • SE l'account è collegato correttamente a Github:
      • Accedo alla sezione "crea repository".
      • Incollo il nome della repo

    • ALTRIMENTI:
      • Accedo al profilo Github inserendo E-mail e Password
      • Accedo alla sezione "crea repository".
      • Incollo il nome della repo

  5. Apro VisualStudio Code.

  6. Clono la repository.

  7. seleziono il file GitHub.

  8. Seleziono la repository location sul computer

    • seleziono la catella degli esercizi.
    • creo una catella all'interno dove inserisco il numero e il nome dell'esercizio.

  9. Comincio a lavorare all'esercizio.

    • SE l'esercizio comprende file allegati:
      • Torno alla sezione "comunicazioni" di Slack e scarico i file da comprendere nell'esercizio.
      • creo cartelle dove inserire i file.
      • inserisco file e immagini all'interno dell'esercizio.

    • ALTRIMENTI:
      • Non inserisco nulla di aggiuntivo all'esercizio.

  10. Immancabilmente metto musica da Youtube

  11. Comincio a lavorare alla macro struttura del progetto.

    • SE l'esercizio consiste nel ricreare una pagina di riferimento:
      • Apro a parte l'immagine del sito da ricreare in modo da avere un punto di riferimento

    • ALTRIMENTI:
      • Comincio a lavorare senza punti di riferimento.

  12. Lavoro al progetto

    • FINCHÉ
      • Finisco l'esercizio
    • O
      • inconto un ostacolo o una difficoltà
        • SE posso risolverla da solo:
          • Mi documento su internet o consulto slide/registrazione della lezione per risolvere il problema

        • ALTRIMENTI:
          • Mi connetto a Slack e richiedo un ticket per capire come risolvere il problema.

  13. FINISCO L'ESERCIZIO.

  14. Commit e Push.
